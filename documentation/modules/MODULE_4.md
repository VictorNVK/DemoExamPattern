# Модуль 4. Работа с заказами

**КОД 09.02.07-2-2026 · М4**

## Задание по ТЗ

1. Кнопка **«Заказы»** для manager и admin.
2. Список заказов по макету из методички.
3. **CRUD заказов** — только **admin**:
   - форма добавления/редактирования;
   - редактирование по клику на элемент;
   - «Добавить заказ» из списка.
4. Поля формы (по ТЗ): **артикул**, **статус** (список), **адрес пункта выдачи**, **дата заказа**, **дата выдачи**.
5. **Удаление** заказа (admin).
6. Обновление списка после CRUD.

---

## Реализация

### Backend API

| Метод | URL | Роль |
|-------|-----|------|
| GET | `/api/orders` | manager, admin |
| GET | `/api/orders/options` | admin |
| GET | `/api/orders/next-id` | admin |
| GET | `/api/orders/{id}` | admin |
| POST | `/api/orders` | admin |
| PUT | `/api/orders/{id}` | admin |
| DELETE | `/api/orders/{id}` | admin |

Сервис: `OrderService`, контроллер: `OrderController`.

Модель `Order`: клиент, менеджер, статус, адрес, даты, товар (артикул через связь с `Product`), количество, цена, скидка.

### Client

| Элемент | Файл |
|---------|------|
| Список заказов | `ui/screens/OrderListScreen.kt` |
| Редактор заказа | `ui/screens/OrderEditorScreen.kt` |
| Модели | `model/OrderModels.kt` |
| Репозиторий | `data/OrderRepository.kt` |
| Навигация | `AppScreen.OrderEditor`, `AppState.openOrderEditor()` |

Кнопка «Заказы» видна при `UserRole.canViewOrders` (manager, admin).  
Добавление/редактирование — при `canManageOrders` / admin.
