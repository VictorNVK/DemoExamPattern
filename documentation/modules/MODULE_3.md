# Модуль 3. UI, поиск, CRUD товаров

**КОД 09.02.07-2-2026 · М3**

## Задание по ТЗ

1. Последовательный UI, кнопка **«Назад»**, заголовки экранов.
2. Обработка ошибок: диалоги (ошибка / предупреждение / информация).
3. **Поиск** (manager, admin) по всем текстовым полям, в реальном времени.
4. **Сортировка** по количеству на складе (возр./убыв.).
5. **Фильтр по поставщику**; первый пункт — «Все поставщики».
6. Поиск + фильтр + сортировка **совместно**, без кнопки «Найти».
7. **CRUD товаров** — только **admin**.
8. Форма редактора: все поля из ТЗ, выпадающие списки, валидация (цена ≥ 0, остаток ≥ 0).
9. ID: при добавлении max+1 (не показывать); при редактировании — только чтение.
10. Фото: загрузка, resize **300×200**, путь в БД, удаление старого файла.
11. Одно окно редактирования одновременно.
12. Удаление: нельзя, если товар в заказе.
13. Обновление списка после CRUD.

---

## Реализация

### Backend API

| Метод | URL | Роль |
|-------|-----|------|
| GET | `/api/products?search&supplierFilter&sortField&sortDirection` | все |
| GET | `/api/products/filter-options` | manager, admin |
| GET | `/api/products/options` | admin |
| GET | `/api/products/next-id` | admin |
| GET/POST/PUT/DELETE | `/api/products/{id}` | admin |
| POST | `/api/products/{id}/image` | admin |

Сервисы: `ProductService`, `ImageStorageService` (resize 300×200).

### Client

| Функция | Файл |
|---------|------|
| Навигация, стек «Назад» | `AppState.kt`, `AppScreen.kt` |
| Фильтры, «Все поставщики» | `ProductListScreen.kt` |
| Редактор товара | `ProductEditorScreen.kt` |
| Валидация | `util/ProductValidation.kt` |
| Диалоги | `ui/components/MessageDialog.kt` |
| REST | `api/BackendClient.kt`, `data/ProductRepository.kt` |

### Дополнительно (сверх минимума ТЗ)

- Фильтр по **диапазону скидки** (удобство для демо).
- Сортировка по **цене** (кроме остатка из ТЗ).
