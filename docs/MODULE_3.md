# Модуль 3. Навигация, поиск, CRUD товаров

Соответствует заданию **М3** (КОД 09.02.07-2-2026): последовательный UI, поиск/фильтрация/сортировка, добавление/редактирование/удаление товаров.

## Требования ТЗ

### Навигация и UX

- Переход между экранами с кнопкой «Назад».
- Заголовок на каждом экране.
- Диалоги ошибок, предупреждений, информации.
- Одно окно редактирования товара одновременно.

### Поиск, фильтр, сортировка (менеджер и администратор)

- **Поиск** по всем текстовым полям товара.
- **Сортировка** по количеству на складе (возр. / убыв.).
- **Фильтр по скидке** (в UI; в ТЗ также упоминается фильтр по поставщику — см. `ProductFilter`).
- Изменения в реальном времени, без кнопки «Найти».
- Поиск и фильтр работают совместно, сортировка сохраняется.

### CRUD товаров (только администратор)

- Добавление и редактирование на отдельной форме.
- Поля: фото, наименование, категория, описание, производитель, поставщик, цена, ед. изм., остаток, скидка.
- ID при добавлении: max(id) + 1; при редактировании — только чтение.
- Загрузка фото: resize **300×200**, хранение в `storage/images/`, путь в БД.
- Удаление запрещено, если товар есть в заказе.

## Реализация

### Backend API

| Метод | URL | Роль |
|-------|-----|------|
| GET | `/api/products?search&discountFilter&sortField&sortDirection` | все |
| GET | `/api/products/{id}` | admin |
| GET | `/api/products/next-id` | admin |
| GET | `/api/products/options` | admin |
| POST | `/api/products` | admin |
| PUT | `/api/products/{id}` | admin |
| DELETE | `/api/products/{id}` | admin |
| POST | `/api/products/{id}/image` | admin |

Логика: `ProductService`, `ImageStorageService`.

**Параметры фильтрации** (`GET /api/products`):

| Параметр | Значения |
|----------|----------|
| search | строка поиска |
| discountFilter | `ALL`, `UP_TO_12_99`, `FROM_13_TO_16_99`, `FROM_17` |
| sortField | `NONE`, `PRICE`, `STOCK` |
| sortDirection | `ASC`, `DESC` |

Справочники для выпадающих списков — уникальные значения из существующих товаров (`/api/products/options`).

### Client

| Функция | Файл |
|---------|------|
| Список + фильтры | `ProductListScreen.kt` |
| Редактор товара | `ProductEditorScreen.kt` |
| Валидация | `util/ProductValidation.kt` |
| REST-клиент | `api/BackendClient.kt` |
| Репозиторий | `data/ProductRepository.kt` |

Навигация: `navigation/AppScreen.kt`, стек экранов в `AppState.kt`.

## Проверка модуля

1. Войти как **admin**.
2. Добавить товар с фото — проверить файл в `storage/images/`.
3. Отредактировать — поля подгружаются из API.
4. Удалить товар без заказов — успех; товар из заказа — ошибка 409.
5. Войти как **manager** — поиск и сортировка доступны, кнопки CRUD скрыты.
6. Кнопка «Назад» возвращает на предыдущий экран.
