# Модуль 4. Заказы

Соответствует заданию **М4** (КОД 09.02.07-2-2026): работа с заказами для менеджера и администратора.

## Требования ТЗ

- Кнопка **«Заказы»** в интерфейсе менеджера и администратора.
- Список заказов по макету из методички.
- **Добавление / редактирование / удаление** — только **администратор** (формы в полной реализации ТЗ; в текущем шаблоне — просмотр списка через API).

### Поля заказа (по ТЗ)

- Артикул (товар)
- Статус заказа
- Адрес пункта выдачи
- Дата заказа, дата выдачи
- Клиент, менеджер, комментарий, код получения

## Реализация

### Backend

| Компонент | Описание |
|-----------|----------|
| `Order` | JPA-сущность, таблица `orders` |
| `OrderService` | Формирование `OrderSummaryDto` |
| `OrderController` | `GET /api/orders` |

**Доступ:** роли `manager`, `admin` (Bearer token).

Пример ответа:
```json
{
  "id": 1,
  "customerName": "Кузнецов Алексей",
  "managerName": "Петров Петр Петрович",
  "statusName": "Новый",
  "orderDate": "2026-05-01T10:30:00",
  "comment": "Позвонить за час",
  "itemsCount": 2,
  "totalAmount": 1618.20
}
```

CRUD заказов (`POST/PUT/DELETE /api/orders`) зарезервирован в `SecurityConfig` для доработки по ТЗ.

### Client

| Компонент | Файл |
|-----------|------|
| Экран заказов | `ui/screens/OrderListScreen.kt` |
| Модель | `model/OrderSummary.kt` |
| Репозиторий | `data/OrderRepository.kt` |

Открытие: `AppState.openOrders()` — доступно при `UserRole.canViewOrders`.

## Связь с удалением товаров

Если товар присутствует в заказе, `DELETE /api/products/{id}` возвращает **409 Conflict** — требование ТЗ модуля 3.

## Проверка модуля

1. Войти как **manager** или **admin**.
2. Нажать «Заказы» — список из БД.
3. Войти как **client** — кнопка недоступна / предупреждение о правах.
4. В Swagger: `GET /api/orders` с токеном manager/admin.
